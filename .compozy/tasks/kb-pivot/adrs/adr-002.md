# ADR-002: Rename Binary to `kb`

## Status

Accepted

## Date

2026-04-11

## Context

The project is pivoting from a codebase-only tool to a general-purpose knowledge base CLI. The name "kodebase" implies a code-only focus that no longer matches the tool's purpose. The binary name appears in every CLI invocation and should be short, memorable, and descriptive of the broader mission.

## Decision

Rename the binary from `kodebase` to `kb`. The Go module path remains `github.com/pedronauck/kodebase` (repo name unchanged), but `cmd/kodebase/` becomes `cmd/kb/` and the compiled binary is `kb`.

## Alternatives Considered

### Alternative 1: Keep `kodebase`

- **Description**: Retain the existing name
- **Pros**: No breaking change, established identity
- **Cons**: Name implies code-only focus, misleading for general KB usage
- **Why rejected**: Name-purpose mismatch grows as more ingest sources are added

### Alternative 2: `kbase`

- **Description**: Shortened compromise — "k" from kodebase/knowledge, "base" from knowledge base
- **Pros**: Slightly more descriptive than `kb`
- **Cons**: Less memorable, not a well-known abbreviation
- **Why rejected**: `kb` is already the universal abbreviation for "knowledge base"

## Consequences

### Positive

- Short, fast to type, universally understood abbreviation
- Matches the command prefix pattern (`kb topic`, `kb ingest`)

### Negative

- Potential collision with other `kb` binaries on some systems
- Breaking change for existing users (minor — project is pre-release)

### Risks

- None significant — project is pre-1.0

## Implementation Notes

- Rename `cmd/kodebase/` to `cmd/kb/`
- Update Makefile build target
- Update Cobra root command `Use` field
- Go module path stays `github.com/pedronauck/kodebase`

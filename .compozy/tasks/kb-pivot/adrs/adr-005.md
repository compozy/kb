# ADR-005: Native Go Vault Lint Engine

## Status

Accepted

## Date

2026-04-11

## Context

The karpathy-kb skill uses a Python script (`lint-wiki.py`) to detect dead wikilinks, orphan articles, and missing source references in the knowledge base vault. As the CLI pivots to be the structural backbone of the KB pattern, lint should be a first-class Go command rather than a Python dependency.

## Decision

Rewrite the lint logic in Go as the `kb lint` command. The command walks the vault directory tree, parses markdown frontmatter with YAML, extracts wikilinks via regex, and runs structural health checks:

1. **Dead wikilinks** — `[[Target]]` links where the target file does not exist
2. **Orphan articles** — wiki articles with zero incoming wikilinks
3. **Missing sources** — frontmatter `sources:` entries referencing absent `raw/` files
4. **Stale content** — articles whose `updated` date is older than their source files' `scraped` date
5. **Format violations** — files missing required frontmatter fields per their type/stage

Output is formatted via the existing `internal/output` formatter (table/json/tsv) and optionally saved to `outputs/reports/<date>-lint.md`.

## Alternatives Considered

### Alternative 1: Keep Python script

- **Description**: Shell-out to `lint-wiki.py` from Go
- **Pros**: Less implementation work, existing tested logic
- **Cons**: Python runtime dependency, subprocess communication overhead, harder to extend
- **Why rejected**: Conflicts with single-binary philosophy; lint is a core CLI feature, not an optional extra

## Consequences

### Positive

- Zero external dependencies for lint
- Structured output via existing formatter (table/json/tsv)
- Testable in Go with fixture vaults
- Can be extended with new checks without Python knowledge

### Negative

- Must reimplement wikilink parsing and frontmatter validation in Go
- Initial implementation effort higher than wrapping the script

### Risks

- Wikilink regex may miss edge cases (nested links, escaped brackets) — mitigate with comprehensive test cases from real vaults

## Implementation Notes

- Wikilink extraction: `\[\[([^\]]+)\]\]` regex, split on `|` for display text
- Frontmatter parsing: split on `---` delimiters, decode with `gopkg.in/yaml.v3`
- File graph: build in-memory map of `path → {incoming, outgoing}` links, then check connectivity
- Report generation: reuse `internal/output.FormatOutput` for terminal display

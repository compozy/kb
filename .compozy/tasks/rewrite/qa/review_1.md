### 1. Happy-path generated content fails `kb lint`

- Severity: high
- Area: ingest/render/lint contract
- Reproduction:
  - `./bin/kb --vault <vault> topic new rewrite-qa "Rewrite QA" engineering`
  - `./bin/kb --vault <vault> ingest bookmarks <bookmarks.md> --topic rewrite-qa`
  - `./bin/kb --vault <vault> ingest file internal/convert/testdata/html/simple.html --topic rewrite-qa`
  - `./bin/kb --vault <vault> ingest codebase internal/generate/testdata/fixture-go-repo --topic rewrite-qa --progress never`
  - `./bin/kb --vault <vault> lint rewrite-qa`
- Actual:
  - `kb lint` reports multiple format errors against content produced by the successful happy path.
  - Examples observed:
    - `raw/bookmarks/ai-links.md` missing `status`, `created`, `updated`, `source_urls`
    - `raw/codebase/files/...`, `raw/codebase/symbols/...`, `raw/codebase/indexes/...` missing `scraped`
    - `wiki/concepts/Kodebase - Circular Dependencies.md` missing `sources`
- Expected:
  - Content generated by first-party `kb ingest ...` and `kb ingest codebase` flows should satisfy the schema enforced by first-party `kb lint`.
- Notes:
  - This is a contract break inside the product, not user error.
  - Frontmatter evidence was confirmed directly in generated files under the temp vault.

### 2. Default hybrid `kb search` is not operational on hosts without sqlite-vec support

- Severity: high
- Area: QMD integration / capability negotiation
- Reproduction:
  - `./bin/kb --vault <vault> --topic rewrite-qa index --embed=false`
  - `./bin/kb --vault <vault> --topic rewrite-qa search greeter`
- Actual:
  - Lexical search works (`kb search greeter --lex`).
  - Default hybrid search delegates to `qmd query` and fails when vector support is unavailable.
  - Direct reproduction of the underlying command returns a `SQLiteError: no such module: vec0`.
- Expected:
  - The default `kb search` path should remain usable on systems where lexical QMD works but vector search is unavailable, or it should fail with a clean, actionable kb-level message instead of exposing the raw QMD stack trace path.
- Notes:
  - The issue was reproduced both through `kb search` and by running `qmd query --json -n 10 -c rewrite-qa greeter` directly.
  - This is a common edge case on hosts without sqlite extension loading / sqlite-vec availability.
